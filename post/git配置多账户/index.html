git 配置多账户

@Jepp Hu · Feb 21, 2019 · 1 min read

1. 为不同的账号生成对应的 key

检查是否有 .ssh 目录

cd ~/.ssh
#如果目录不存在，执行
ssh-keygen

生成一个 github 的 key

ssh-keygen -t rsa -C "youremail@domain.com"
#执行命令后的第一次输入，输入一个用于区分的自定义名字,不要有空格，如：my_github_rsa， 回车
第二次与第三次的输入是为了设定访问 key 的密码，可以直接回车跳过

生成一个 bitbucket 的 key

ssh-keygen -t rsa -C "youremail@domain.com"
#同上自定义名字，如：my_bitbucket_rsa

2. 修改配置文件

#进入 .ssh 目录，如果目录下没有 config 文件，那么创建
cd ~/.ssh
touch config

#添加以下内容
Host defaultHost
    HostName github.com
    PreferredAuthentications publickey
    IdentityFile ~/.ssh/my_github_rsa
Host secondHost
    HostName bitbucket.org
    PreferredAuthentications publickey
    IdentityFile ~/.ssh/my_bitbucket_rsa

3. 向 github 和 bitbucket 添加密钥

此时在 .ssh 目录下至少会有以下5个文件

config           
my_bitbucket_rsa    my_github_rsa
my_bitbucket_rsa.pub    my_github_rsa.pub

.pub 后缀的为公钥，添加在 github 或者 bitbucket 中， 以 github 为例：

  1. github 首页右上角头像下拉框 –> settings –> SSH and GPG keys
  2. title 部分，用于区分设备的名称
  3. key 部分，复制 my_github_rsa.pub 中的内容

4. 测试

ssh -T git@github.com
#输出 Hi XXX! You've successfully authenticated, but GitHub does not provide shell access.

ssh ssh -T git@bitbucket.org
#输出 You can use git or hg to connect to Bitbucket. Shell access is disabled

#测试过程中出现 The authenticity of host 'github.com (192.xxx.xxx.xxx)' can't be established.
RSA key fingerprint is SHA256:...   
输入 yes，回车
