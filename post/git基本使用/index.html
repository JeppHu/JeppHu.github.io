git 基本使用

@Jepp Hu · Apr 2, 2018 · 1 min read

Git 配置用户名和邮箱

配置全局用户名和邮箱，不适用多账户

git config --global user.name "your name"
git config --global user.email "your eamil@domain.com"

查看用户名和邮箱

git config user.name
git config user.email

解除全局用户名和邮箱

git config --global --unset user.name
git config --global --unset user.email

查看 git 所有配置

git config --list

上传本地项目到远程仓库

  1. 建立一个远程仓库 : git@github.com:xxx/xxx.git
  2. 进入项目根目录
  3. 初始化 git，并与远程仓库建立联系，将本地项目提交到远程仓库
git init
git remote add origin git@github.com:xxx/xxx.git
git add -A
git commit -m "init"
git push origin master

克隆远程仓库的项目到本地

  1. 进入准备存放项目的根目录
  2. 将远程项目拉取到本地
git clone git@github.com:xxx/xxx.git

项目管理

提交修改
  1. 将新加文件添加到暂存区
  2. 将本地修改提交到工作区
  3. 将本地库中的最新 commit 发送给远程库，确认到具体分支
git add -A
git commit -a -m "描述"
git push origin master

拉取修改，从远程获取最新版本到本地并从指定的 commit 合并到当前分支

git pull origin master
#或
git fetch
git merge

查看前三个 commit 的版本

git log -3
回滚代码
  • 回滚到指定的版本，此版本之后的所有版本被保留，文件内容保留，回滚 commit 和 index 信息，相当于回到了没有 add 和 commit 的状态
git reset 9cac9ba76574da2167
或 
git reset --mixed 9cac9ba76574da2167
  • 回滚到指定的版本，此版本之后的所有版本被保留，并且文件内容保留，回滚 commit 信息
git reset --soft 9cac9ba76574da2167
  • 回滚到指定的版本，但是此版本之后的所有版本都会被删除，慎用
git reset --hard 9cac9ba76574da2167
  • reset 回滚是不会删除新添加的文件，如果要删除
git clean -df

分支管理

查看本地分支，查看所有分支

git branch
git branch -a

新建一个分支，切换到新分支

git branch branchName
git checkout branchName
#或
git checkout -b branchName

删除本地分支

git branch -D branchName

删除远程分支

git branch -r -d origin/branchName
git push origin :branchName

从远程获取最新版本到本地，不会自动 merge

git fetch

清理远程分支列表中已删除的分支

git fetch origin --prune

拉取远程分支到本地并与远程的分支建立映射关系

git checkout 远程分支名
#或
git checkout -b 本地分支名 origin/远程分支名

源管理

更换项目对应的远程仓库的源

  1. 查看源
  2. 移除本来的源
  3. 添加新的源
git remote -v
git remote remove origin
git remote add origin git@github.com:xxx/xxx.git
